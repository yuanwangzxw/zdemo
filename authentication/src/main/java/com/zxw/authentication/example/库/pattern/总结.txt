##增强型
适配，装饰，代理
类与域属性实现不同接口：适配，同一接口：装饰，代理
代理：不需要外部传入源对象，一般不关心源对象是什么，有访问控制，装饰：灵活组合，一般会传入源对象做增强

##组合型
桥接，装饰，组合
实现不同接口：桥接，同一接口：装饰
关联关系：对象（桥接，装饰）集合（组合）

##选择型
策略，状态，责任链，命令
策略：一次请求只会走一种策略，责任链：一次请求有可能命中多个节点
责任链特别像装饰者：都是类里面包含同类型的域属性，在方法里调用域属性同样的方法，都是组合的顺序不同来实现整体逻辑
命令：类似于事件驱动，将请求封装成事件，由事件分发器发到不同的handler处理

##复用型
单例，享元
享元，用缓存存入属性大部分相同的对象，比如棋子只有座标会变，商品只有库存会变，
缓存取出对象，实时查询会变动的值，池中的对象不可修改，会有线程安全问题
